Определю объем и артефакты. Поскольку пользователь хочет "правдивую постановку задачи на аутентификацию", я создам комплекс артефактов для типовой системы аутентификации с использованием современных практик (JWT, OAuth2, многофакторная аутентификация). Будут созданы:

1. Backend logic description (описание логики аутентификации)
2. ERD для данных пользователей, сессий, токенов
3. Sequence diagram для процессов входа, регистрации, обновления токена
4. OpenAPI спецификация для REST API аутентификации
5. AsyncAPI спецификация для событий аутентификации (например, события "пользователь зарегистрирован")
6. Non-functional requirements для безопасности, производительности и надежности
