## UC-01: Перевод на карту другого лица по номеру карты

| Элемент | Описание |
|---------|----------|
| **Название** | UC-01. Перевод на карту другого лица по номеру карты |
| **User Story** | [`US-001`](card2card_transfer_us.md#us-001-перевод-по-номеру-карты): Как клиент банка, я хочу перевести деньги с моей карты на карту другого человека по номеру карты, чтобы быстро и удобно отправлять средства |
| **Участники** | • **Основной:** Отправитель (физическое/юридическое лицо)<br>• **Вторичные:** Мобильное приложение / Интернет-банк, Core Banking System, Платежный шлюз, Антифрод-система, Система уведомлений |
| **Предусловия** | • Отправитель авторизован в системе<br>• У отправителя есть действующая карта с достаточным остатком<br>• Номер карты получателя введен и прошел базовую валидацию |
| **Ограничения** | • Номер карты: 16-19 цифр<br>• Сумма перевода не превышает настраиваемые лимиты<br>• Поддержка карт Visa, Mastercard, МИР |
| **Триггер** | Отправитель нажимает кнопку «Перевести по карте» в мобильном приложении или интернет-банке |
| **Основной сценарий** | 1. Отправитель выбирает опцию «Перевод по номеру карты»<br>2. Система отображает форму: выбор карты списания, номер карты получателя, сумма<br>3. Отправитель вводит номер карты получателя (16-19 цифр)<br>4. Система валидирует номер карты (алгоритм Луна)<br>5. Система определяет платежную систему и банк-эквайер получателя<br>6. Отправитель вводит сумму перевода<br>7. Система проверяет лимиты (суточные, месячные, по валюте)<br>8. Система проверяет достаточность средств на карте списания<br>9. Система рассчитывает комиссию (если применимо)<br>10. Система отображает детали перевода для подтверждения (сумма, комиссия, маскированная карта получателя)<br>11. Отправитель подтверждает перевод (push/SMS/OTP)<br>12. Система передает транзакцию в платежный шлюз<br>13. Платежный шлюз обрабатывает перевод (on-us или off-us)<br>14. Система списывает средства с карты отправителя<br>15. Система зачисляет средства на карту получателя<br>16. Система отправляет уведомления: отправителю — об успешном списании, получателю — о зачислении<br>17. Система отображает отправителю экран успешного перевода |
| **Альтернативные сценарии** | **п.4**: Номер карты не прошел валидацию → показать сообщение «Неверный номер карты»<br>**п.7**: Превышен лимит → показать сообщение «Превышен лимит. Максимальная сумма: X»<br>**п.8**: Недостаточно средств → показать сообщение «Недостаточно средств на карте»<br>**п.11**: Ошибка подтверждения (неверный код) → предложить повторить (до 3 попыток), затем блокировка на 30 минут<br>**п.13**: Платежный шлюз недоступен → показать сообщение «Сервис временно недоступен, попробуйте позже»<br>**п.13**: Ошибка обработки транзакции (4XX/5XX) → показать сообщение «Ошибка обработки перевода. Средства не списаны»<br>**п.15**: Карта получателя заблокирована → вернуть средства отправителю, уведомить об ошибке |
| **Постусловия** | • Средства списаны с карты отправителя<br>• Средства зачислены на карту получателя (on-us — мгновенно, off-us — в соответствии с регламентом ПС)<br>• Транзакция сохранена в истории операций<br>• Уведомления отправлены обеим сторонам<br>• Метрики операции отправлены в систему мониторинга |

## UC-02: Перевод между своими картами (Me-to-Me)

| Элемент | Описание |
|---------|----------|
| **Название** | UC-02. Перевод между своими картами |
| **User Story** | [`US-002`](card2card_transfer_us.md#us-002-перевод-между-своими-картами-me-to-me): Как клиент банка, я хочу переводить деньги между моими собственными картами, чтобы управлять остатками на разных счетах без комиссии |
| **Участники** | • **Основной:** Отправитель (клиент банка)<br>• **Вторичные:** Мобильное приложение / Интернет-банк, Core Banking System, Система уведомлений |
| **Предусловия** | • Отправитель авторизован в системе<br>• У отправителя есть минимум 2 действующие карты в банке<br>• На карте-источнике достаточно средств |
| **Ограничения** | • Перевод только между картами одного клиента<br>• Бесплатный тариф<br>• Мгновенное зачисление (on-us) |
| **Триггер** | Отправитель выбирает опцию «Перевод между своими картами» |
| **Основной сценарий** | 1. Отправитель выбирает опцию «Перевод между своими картами»<br>2. Система отображает список всех карт клиента с остатками<br>3. Отправитель выбирает карту-источник (счет списания)<br>4. Отправитель выбирает карту-назначение (счет зачисления)<br>5. Отправитель вводит сумму перевода<br>6. Система проверяет достаточность средств<br>7. Система отображает детали: карта списания, карта зачисления, сумма, комиссия = 0 ₽<br>8. Отправитель подтверждает перевод<br>9. Система выполняет внутреннюю проводку (on-us)<br>10. Система отображает экран успешного перевода<br>11. Система отправляет push-уведомление об операции |
| **Альтернативные сценарии** | **п.6**: Недостаточно средств → показать сообщение «Недостаточно средств на карте-источнике»<br>**п.8**: Ошибка подтверждения → повторить попытку (до 3 раз)<br>**п.9**: Сбой Core Banking System → показать сообщение «Ошибка обработки, попробуйте позже» |
| **Постусловия** | • Средства переведены между картами клиента<br>• Операция отражена в истории<br>• Отправлено уведомление |

## UC-03: Мультивалютный перевод на карту другого лица

| Элемент | Описание |
|---------|----------|
| **Название** | UC-03. Мультивалютный перевод на карту другого лица |
| **User Story** | [`US-003`](card2card_transfer_us.md#us-003-мультивалютный-перевод): Как клиент банка, я хочу переводить деньги с карты в одной валюте на карту в другой валюте, чтобы отправлять средства за границу или конвертировать валюту |
| **Участники** | • **Основной:** Отправитель<br>• **Вторичные:** Мобильное приложение / Интернет-банк, Core Banking System, Платежный шлюз, Система валютного контроля, Система уведомлений |
| **Предусловия** | • Отправитель авторизован<br>• У отправителя есть карта в валюте, отличной от валюты карты получателя<br>• Курс конвертации доступен |
| **Ограничения** | • Поддерживаемые валюты: RUB, USD, EUR<br>• Сумма свыше 10 000 USD подлежит валютному контролю<br>• Комиссия за конвертацию + комиссия за перевод |
| **Триггер** | Отправитель вводит номер карты получателя в валюте, отличной от валюты своей карты |
| **Основной сценарий** | 1. Отправитель выбирает опцию «Перевод по номеру карты»<br>2. Система определяет валюту карты получателя (отличную от валюты карты отправителя)<br>3. Система отображает: текущий курс конвертации, сумма списания, сумма зачисления<br>4. Отправитель вводит сумму в валюте списания или зачисления<br>5. Система пересчитывает сумму по текущему курсу<br>6. Система проверяет лимиты с учетом валютного контроля<br>7. Отправитель подтверждает перевод<br>8. Система выполняет конвертацию по курсу банка<br>9. Система отправляет перевод через платежный шлюз<br>10. Система отправляет уведомления с деталями конвертации |
| **Альтернативные сценарии** | **п.6**: Сумма превышает порог валютного контроля → запросить дополнительные документы (назначение платежа, подтверждение)<br>**п.8**: Курс изменился с момента отображения → показать новый курс и запросить повторное подтверждение<br>**п.9**: Ошибка валютного контроля → уведомить отправителя о необходимости предоставить документы |
| **Постусловия** | • Конвертация выполнена по курсу банка<br>• Средства зачислены в валюте карты получателя<br>• Операция зарегистрирована в системе валютного контроля (если применимо) |

## UC-04: Просмотр истории переводов

| Элемент | Описание |
|---------|----------|
| **Название** | UC-04. Просмотр истории переводов |
| **User Story** | [`US-004`](card2card_transfer_us.md#us-004-просмотр-истории-переводов): Как клиент банка, я хочу просматривать историю всех моих переводов с фильтрацией и поиском, чтобы контролировать свои расходы |
| **Участники** | • **Основной:** Клиент (отправитель/получатель)<br>• **Вторичные:** Мобильное приложение / Интернет-банк, Core Banking System |
| **Предусловия** | • Клиент авторизован<br>• У клиента есть хотя бы одна завершенная операция перевода |
| **Ограничения** | • Отображение операций за выбранный период<br>• Фильтрация по дате, сумме, статусу, карте |
| **Триггер** | Клиент переходит в раздел «История переводов» |
| **Основной сценарий** | 1. Клиент открывает раздел «История переводов»<br>2. Система отображает список операций (по умолчанию — за последние 30 дней)<br>3. Клиент может применить фильтры: дата, сумма, статус, карта<br>4. Клиент выбирает конкретную операцию<br>5. Система отображает детали: дата, сумма, комиссия, статус, карты, номер транзакции<br>6. Клиент может экспортировать историю в PDF или CSV |
| **Альтернативные сценарии** | **п.2**: Операции отсутствуют → показать сообщение «У вас пока нет переводов»<br>**п.3**: Поиск по номеру карты не дал результатов → показать «Ничего не найдено»<br>**п.6**: Ошибка экспорта → показать сообщение «Не удалось сформировать файл, попробуйте позже» |
| **Постусловия** | • Клиент получил информацию об операциях<br>• При экспорте — файл сохранен на устройство |

## UC-05: Работа с шаблонами переводов

| Элемент | Описание |
|---------|----------|
| **Название** | UC-05. Работа с шаблонами переводов |
| **User Story** | [`US-005`](card2card_transfer_us.md#us-005-шаблоны-переводов): Как клиент банка, я хочу сохранять реквизиты частых получателей как шаблоны, чтобы не вводить номер карты каждый раз |
| **Участники** | • **Основной:** Клиент<br>• **Вторичные:** Мобильное приложение / Интернет-банк, База данных шаблонов |
| **Предусловия** | • Клиент авторизован<br>• Выполнен хотя бы один успешный перевод |
| **Ограничения** | • Не более 50 шаблонов на пользователя<br>• Шаблон содержит: название, номер карты (маскированный), банк получателя |
| **Триггер** | Клиент нажимает «Сохранить как шаблон» после успешного перевода или открывает раздел «Шаблоны» |
| **Основной сценарий** | 1. Клиент выполняет перевод и нажимает «Сохранить как шаблон»<br>2. Система предлагает ввести название шаблона<br>3. Клиент вводит название и подтверждает сохранение<br>4. Система сохраняет шаблон<br>5. При следующем переводе клиент выбирает шаблон из списка<br>6. Система автоматически подставляет номер карты получателя |
| **Альтернативные сценарии** | **п.1**: Достигнут лимит шаблонов (50) → показать сообщение «Достигнут лимит шаблонов. Удалите неиспользуемые»<br>**п.3**: Название не введено → использовать маскированный номер карты как название по умолчанию<br>**п.5**: Шаблон содержит карту, по которой был возврат → пометить шаблон как «требует проверки» |
| **Постусловия** | • Шаблон сохранен и доступен для быстрых переводов<br>• Шаблон можно редактировать или удалить |

## UC-06: Подтверждение перевода

| Элемент | Описание |
|---------|----------|
| **Название** | UC-06. Подтверждение перевода |
| **User Story** | [`US-006`](card2card_transfer_us.md#us-006-подтверждение-перевода): Как клиент банка, я хочу подтверждать перевод через push-уведомление или SMS, чтобы быть уверенным в безопасности операции |
| **Участники** | • **Основной:** Отправитель<br>• **Вторичные:** Мобильное приложение, SMS-шлюз, Система аутентификации |
| **Предусловия** | • Отправитель заполнил все поля перевода<br>• Система отобразила детали для подтверждения |
