## 1. Методология сбора

Сбор стейкхолдеров выполнен на основе анализа:
- Бизнес-требований (BRD) услуги «Перевод с карты на карту»
- Организационной структуры типового банка
- Категорий стейкхолдеров по BABOK v3

## 5. Рекомендации по коммуникации

### 5.1. Регулярные встречи
| Встреча | Участники | Периодичность |
|---------|-----------|---------------|
| Steering Committee | Спонсор, Владелец продукта, Руководитель ИТ | Ежемесячно |
| Daily Standup | Команда разработки, Владелец продукта, QA | Ежедневно |
| Архитектурный комитет | Системный архитектор, Руководитель ИТ, Платежные системы | Еженедельно |
| Демо (Sprint Review) | Все стейкхолдеры | Раз в 2 недели |
| Комплаенс-ревью | Комплаенс, Юрист, CISO, Владелец продукта | Ежемесячно |

### 5.2. Каналы коммуникации
- **Email-рассылка:** Еженедельный статус-репорт для всех стейкхолдеров
- **Jira/Confluence:** Доступ к бэклогу и документации для технических стейкхолдеров
- **Slack/Teams:** Оперативные каналы для команды разработки
- **Презентации:** Для спонсоров и руководства — ежемесячные обзоры

## 6. Риски, связанные со стейкхолдерами

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Изменение приоритетов со стороны спонсора | Средняя | Высокое | Фиксация требований в BRD, регулярные демо |
| Задержки согласования комплаенс | Высокая | Среднее | Параллельное согласование, раннее вовлечение |
| Конфликт требований розничного и корпоративного бизнеса | Средняя | Среднее | Приоритизация через Product Owner |
| Недостаточное вовлечение маркетинга | Низкая | Среднее | Включение маркетинга в регулярные демо |
