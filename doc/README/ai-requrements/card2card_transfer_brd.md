# Бизнес-требования (BRD): Услуга «Перевод с карты на карту»

**Версия:** 1.0  
**Дата:** 24.04.2026  
**Статус:** Черновик  
**Автор:** Бизнес-аналитик  

---

## 1. Цель документа

Настоящий документ определяет бизнес-требования к созданию новой банковской услуги «Перевод денежных средств с карты на карту» (Card-to-Card Transfer). Документ предназначен для согласования с заинтересованными сторонами и передачи системному аналитику для дальнейшей детализации.

---

## 2. Бизнес-контекст и обоснование

### 2.1. Предпосылки создания услуги
- Растущий спрос клиентов на мгновенные переводы между картами
- Наличие аналогичных услуг у банков-конкурентов (Сбер, Т-Банк, Альфа-Банк)
- Потребность в расширении продуктовой линейки банка для удержания и привлечения клиентов
- Возможность получения комиссионного дохода от переводов на карты сторонних банков

### 2.2. Бизнес-цели
1. **Увеличение клиентской базы** — привлечение новых клиентов за счет удобного сервиса переводов
2. **Повышение лояльности** — удержание существующих клиентов через бесплатные переводы между своими картами
3. **Дополнительный доход** — получение комиссионного вознаграждения за переводы на карты сторонних банков
4. **Конкурентное преимущество** — выход на рынок с мультивалютными переводами

### 2.3. Ключевые показатели успеха (KPI)
| Показатель | Целевое значение |
|------------|-----------------|
| Количество переводов в месяц | Не менее 50 000 в первый месяц |
| Доля успешных переводов | ≥ 99% |
| Среднее время зачисления | ≤ 30 секунд |
| NPS (Net Promoter Score) | ≥ 70 |
| Доля клиентов, использующих услугу | ≥ 15% от активной клиентской базы |

## 4. Функциональные требования

### 4.1. Основной функционал

#### FR-01: Перевод по номеру карты
**Описание:** Возможность осуществлять перевод денежных средств с карты отправителя на карту получателя по номеру карты (PAN).

**Требования:**
- Ввод номера карты получателя (16-19 цифр)
- Автоматическое определение платежной системы (Visa, Mastercard, МИР)
- Валидация номера карты (алгоритм Луна)
- Отображение маскированного номера карты получателя для подтверждения
- Поддержка переводов как на карты банка, так и на карты сторонних банков

#### FR-02: Перевод между своими картами (Me-to-Me)
**Описание:** Возможность перевода между собственными картами клиента, открытыми в банке.

**Требования:**
- Отображение списка всех карт клиента
- Выбор карты-источника и карты-назначения
- Мгновенное зачисление (on-us переводы)
- Бесплатный тариф

#### FR-03: Мультивалютные переводы
**Описание:** Поддержка переводов в различных валютах.

**Требования:**
- Поддерживаемые валюты: RUB, USD, EUR (с возможностью расширения)
- Автоматическая конвертация валюты при несовпадении валют карт
- Отображение курса конвертации и суммы списания/зачисления до подтверждения
- Поддержка валютного контроля (для сумм свыше эквивалента 10 000 USD)

#### FR-04: История операций
**Описание:** Просмотр истории всех переводов.

**Требования:**
- Отображение списка всех переводов с фильтрацией по дате, сумме, статусу
- Детальная информация по каждому переводу (дата, сумма, комиссия, статус, карты)
- Экспорт истории в PDF/CSV
- Поиск по номеру карты получателя

#### FR-05: Шаблоны переводов
**Описание:** Сохранение часто используемых реквизитов для быстрых переводов.

**Требования:**
- Сохранение карты получателя как шаблона
- Возможность задать название шаблона
- Быстрый перевод по сохраненному шаблону
- Редактирование и удаление шаблонов
- Ограничение: не более 50 шаблонов на пользователя

#### FR-06: Подтверждение перевода
**Описание:** Многофакторное подтверждение операции.

**Требования:**
- Подтверждение через Push-уведомление (мобильное приложение)
- Подтверждение через SMS-код
- Подтверждение через одноразовый пароль (OTP)
- Отображение деталей перевода перед подтверждением (сумма, комиссия, карты, курс)

#### FR-07: Уведомления
**Описание:** Информирование пользователя о статусе перевода.

**Требования:**
- Push-уведомление об успешном переводе
- Push-уведомление о получении перевода
- SMS-уведомление (опционально)
- Email-уведомление (для юрлиц)
- Уведомление об ошибке с указанием причины

## 5. Нефункциональные требования

### 5.1. Производительность

| Требование | Значение |
|------------|----------|
| Время обработки перевода (on-us) | ≤ 5 секунд |
| Время обработки перевода (сторонний банк) | ≤ 30 секунд |
| Время отклика интерфейса | ≤ 2 секунды |
| Пропускная способность | ≥ 1000 транзакций в секунду |
| Доступность сервиса | 99.9% (за исключением планового ТО) |

### 5.2. Безопасность

- Шифрование данных при передаче (TLS 1.3)
- Шифрование PAN-данных в покое (AES-256)
- Соответствие стандарту PCI DSS
- Антифрод-мониторинг в реальном времени
- Ограничение количества неудачных попыток (3 попытки, блокировка на 30 минут)
- Двухфакторная аутентификация для подтверждения операций

### 5.3. Надежность

- Отказоустойчивая архитектура (кластеризация)
- Резервное копирование данных (RPO ≤ 15 минут)
- Время восстановления (RTO ≤ 1 час)
- Graceful degradation при недоступности внешних систем

### 5.4. Масштабируемость

- Горизонтальное масштабирование компонентов
- Поддержка роста до 10 000 000 транзакций в месяц
- Возможность добавления новых валют без изменения архитектуры

## 7. Тарифная политика

| Тип перевода | Тариф |
|-------------|-------|
| Перевод между своими картами (me-to-me) | **Бесплатно** |
| Перевод на карту другого клиента банка | **Бесплатно** |
| Перевод на карту стороннего банка | Комиссия (настраиваемая, фиксированная или процентная) |
| Мультивалютный перевод | Комиссия за конвертацию + комиссия за перевод |

> **Примечание:** Тарифы должны быть настраиваемыми через административную панель без изменения кода.

## 8. Лимиты

Система должна поддерживать настраиваемые лимиты (настраиваются через административную панель):

| Тип лимита | Описание |
|------------|----------|
| Лимит на один перевод | Максимальная сумма одной операции |
| Суточный лимит | Максимальная сумма всех переводов за день |
| Месячный лимит | Максимальная сумма всех переводов за месяц |
| Количество переводов в день | Максимальное количество операций в день |
| Лимиты по валютам | Отдельные лимиты для каждой валюты |

> **Примечание:** Значения лимитов определяются на этапе проектирования и должны учитывать требования ЦБ РФ.

## 9. Нормативные требования и комплаенс

### 9.1. Законодательные требования

| Нормативный акт | Требование |
|----------------|------------|
| **115-ФЗ** (ПОД/ФТ) | Идентификация клиента, мониторинг операций, пороговые суммы |
| **161-ФЗ** (О национальной платежной системе) | Обеспечение бесперебойности, защита информации |
| **Указания ЦБ РФ** | Лимиты переводов, отчетность, валютный контроль |
| **152-ФЗ** (О персональных данных) | Обработка и защита персональных данных |
| **PCI DSS** | Защита данных держателей карт |

### 9.2. Требования комплаенс

- **Идентификация:** Проверка отправителя и получателя (для сумм свыше 15 000 RUB)
- **Мониторинг:** Автоматический анализ транзакций на предмет подозрительной активности
- **Блокировка:** Возможность блокировки подозрительных переводов до выяснения обстоятельств
- **Отчетность:** Формирование отчетности для ЦБ РФ и Росфинмониторинга
- **AML-проверки:** Интеграция с системами AML (Anti-Money Laundering)

## 11. Ограничения и допущения

### 11.1. Ограничения
- Услуга доступна только для клиентов банка (отправитель)
- Получатель может быть клиентом любого банка
- Максимальная сумма перевода ограничена лимитами ЦБ РФ и настраиваемыми лимитами
- Переводы в иностранной валюте подлежат валютному контролю

### 11.2. Допущения
- Клиент имеет действующую карту в банке
- Клиент прошел идентификацию (для физлиц — упрощенную или полную)
- Для юрлиц — наличие расчетного счета и корпоративной карты
- Техническая инфраструктура банка поддерживает требуемые интеграции
